Files in: /home/htdocs/cgi-bin/Mustang/VeriCool/
File Description
VeriCoolVareables.pl All vareables for the VeriCool cgi-scripts are defined in this file. All the other scripts executes this script to get all the paths and filenames etc. The file is full of "extras" and you don't have to change everything if the script is used for another project. All the vareables are discribed in the file.
PrintFile.pl This cgi-script prints contents of the file in @ARGV[0] starting from @ARGV[1] ending at @ARGV[2]. If @ARGV[2] = "" then the printing of file ends at EOF. If @ARGV[1] = "" the printing starts at the beginning of the file.

ViewBlockInfo.pl This cgi-script prints out information about a block.
ViewCHFiles.pl Cgi-script that prints out which files that has been changed in the current release. It also puts a link to the comment for that paricular change.
Files in: /home/htdocs/cgi-bin/Mustang/VeriCool/TR/
File Description
AdvancedSort.pl This script generates a HTML form for the advanced TR'sort.
GenFollowUpForm.pl This file generates a HTML form for posting a follow up on an TR.
GenIssueForm.pl This file generates a HTML form for posting a new TR.
Report.pl This scripts is called from the GenIssueForm.pl and GenFollowUpForm.pl when posting a new TR, or a follow up on an old TR. It parses the information into a new (if new tr) databasefile or into an existing databasefile. (if follow up or finished).
ViewStat.pl Generates a chart displaying a summary of all the TR's reported.
ViewTR.pl The viewer for the TR's. It is able to run in several modes. It generates a html page containing all the error reports matching the criteria. The criteria can be one of the three sort options given as @ARGV[1], or criteria selected in the AdvancedSort.pl form. If the sort is advanced the criteria is stored in the $FORM{} hash. The @ARGV[0] contains either "Archive" or "Issue"- that is the directory to look in. @ARGV[0] may also contain Admin. That means that the script shall be run in Admin mode. It allows the site administrator to delete issues. This must be done once in a while to clean up the archive. While this mode must be password protected (.htaccess protection for the directory containing the TRAdmin.pl file is enough) this mode is not yet implemented.

The db files in: /home/htdocs/cgi-bin/Mustang/VeriCool/TR/Issue/ and /home/htdocs/cgi-bin/Mustang/VeriCool/TR/Archive/
File Description
ex. issue_0021.db Thease files are built up by perlscripts after a predefined pattern. Each row in the file is one post in an issue. Ex. the first post is on the first row in the file and the first follow up post is on the second row in the file.
The db files uses the followin pattern:
Status###Block###Release###Written by###Priority###Written weekday###Written month###Written day###Written time###Written year###Header###Error description.
One row for one post.

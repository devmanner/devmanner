Help on TR

General

     The TR system for VeriCool is a totally web based TR system created by Tomas Mannerstedt. It is integrated in the VeriCool Website. You can use The VeriCool Website without the TR system but you can't (without modifications) use the TR system without VeriCool.
     You can set up the $sendmail vareable in the /home/htdocs/cgi-bin/Mustang/VeriCool/TR/Report.pl script. If 1 it send mail to responsible persson, if 0 i doesn't.

Reporting a Trouble

     To report a trouble click "Compose new TR" under "TR" on the menubar. A HTML form will appear on the screen. Fill out the form. All fields must be filled out. Post the TR by clicking "Report". To clear the form and start over click "Clear Form".
     If a release or a block is selected in the VeriCool view tree tat perticular block / release will be pre selected.

Posting a follow up

     To post a follow up click the link below the TR. A HTML form, simular to the "Compose new TR" form, will be displayed. All the fields in the form will must be filled out. If the status is set to "Finish", the TR will be moved to the TR Archive. It will be found under the menu with the same name in the menubar.

Viewing TR's

     To view the TR's use one of the "Sort Descending", "Sort Ascending", "View Latest 10" and "Advanced Sort" under TR in the menubar. The Advanced Sort will be explained in a later chapter. "Sort Descending", "Sort Ascending" can also be found under "TR Archive" in the menubar.
     "Sort Descending": Displays all the TR's (not archive) sorting descending on date.
     "Sort Ascending": Displays all the TR's (not archive) sorting ascending on date.
     "View Latest 10": Displays the latest 10 reported TR's in descending sort.

Advanced Sort

     The advanced sort allows you to view TR's matching criteria you fill out in the Advanced Sort form. The boolean operator is OR. We're working on an AND option, but it's not finished yet...

TR-Admin

     This option allowd user to delete posted TR's. This script must be protected by some password protection. .htaccess protection is enough. This option is not implemented yet since we've had some problems with the .htaccess. The Admin mode is set by calling the ViewTR.pl script with the argument Admin.
